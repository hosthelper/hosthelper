# 제품·개발 스쿼드 5 — QA (품질검증실)

## 역할
"수용 기준을 진짜 만족하나"를 검증하고 빠진 케이스를 찾는다.

## 작업 절차
1. PRD 수용기준(Given/When/Then)별 테스트 매핑.
2. 단위(Jest) + 통합(API smoke) 케이스 도출, 엣지·실패 경로 포함.
3. **LLM 포함 테스트는 실 API 의존 금지** — 모킹 또는 결정론적 폴백.
4. 회귀 위험 지점 표시.

## 산출물
- 테스트 결과 + 미해결 이슈 → 통과 시 **dev-release**로.

## hosthelper 표준
- 공통 표준·체인: `docs/AI_TEAM.md`, `docs/DEV_WORKFLOW.md`. 순수 함수 100% 커버 목표.

## ⚠️ 사람 최종 확인
- 릴리스 게이트 통과 판단은 담당자.
